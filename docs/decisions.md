# Decisions

- 2026-07-17 ‚Äî Worktree isolation is **opt-in per terminal**, not the default. Martin deliberately runs multiple agents in one checkout; the main checkout keeps the dev servers/ports. Rejected: default-isolation, concurrency badge, per-worktree dev servers / port offsetting.
- 2026-07-17 ‚Äî Worktrees are headless scratch buffers: merge back into the main checkout to see anything running. Setup command (per-project, user-authored) instead of auto-copying gitignored files ‚Äî secret-leak risk.
- 2026-07-17 ‚Äî Permission mode is a **per-section Chewo setting** (`--permission-mode` / `--ask-for-approval` per spawn), not `permissions.defaultMode` in `~/.claude/settings.json`. Why: the global route widens every repo on the machine including fresh untrusted clones, and can't cover Codex without hand-writing `config.toml` (forbidden ‚Äî live-rewritten). Diagnosis: neither CLI remembers the mode you flip to, so Chewo's many fresh sessions always started at the default.
- 2026-07-17 ‚Äî Home is a first-class section, not a fallback. Sessions map to their owning section only; nothing inherits the currently-selected project.
- 2026-07-17 ‚Äî Notes workflow (SPEC-NOTES.md): STT runs in a **local sidecar process** with a pluggable JSON-stdio protocol (WhisperKit CLI first, parakeet-mlx later). Rejected: whisper.cpp in-process (rewrites proven confirm-and-seek logic, blocks Parakeet A/B), cloud STT (cost, audio leaves machine).
- 2026-07-17 ‚Äî Notes are **markdown files + folders** (Subject/Topic/note.md) under a configurable root. Rejected: projects.json blob (transcripts bloat it), SQLite (breaks the no-DB/no-cache design).
- 2026-07-17 ‚Äî Notes intelligence (structuring + Q&A) runs via **headless `claude -p`**, Q&A scoped by setting cwd to the subject/topic folder with Read/Grep/Glob only. Rejected: direct Anthropic API (second billing/key path), Ollama (weak long-transcript structuring).
- 2026-07-17 ‚Äî Structuring runs **once on stop** over the full transcript; taxonomy is **fully manual** (user names subject + topic before capture); raw **transcript is kept, audio discarded** after transcription.
- 2026-07-18 ‚Äî Umbrella term for one item in the tab bar is **"session"** (vocabulary only, no code rename yet): what you create/focus/close. Its layers keep their code-level names ‚Äî **tab** (UI chip), **terminal** (`termId`, pty runtime), **conversation** (`sessionId`, the CLI transcript a session may be bound to; shell sessions have none). Rejected "tab" (pure chrome, says nothing about the work) and "terminal" (undersells the agent/dormant layers).
- 2026-07-18 ‚Äî Closing the focused session focuses its **left neighbour** in the same section (fall back to the right, then empty). Previously dropped straight to the empty state even with other sessions open.
- 2026-07-18 ‚Äî Tab bar is a **faithful Chrome replica** (`min-height: 38px`): borderless tabs, packed with 1px `::before` **dividers** between them (hidden either side of the active/hovered tab), not gaps. All tabs are the **same height** ‚Äî the focused tab (`terminal-tab-active`, exists only when `view.kind === 'terminal'`) merely fills with the terminal surface (`--bg`), keeps rounded-top corners, and drops 1px (`margin-bottom: -1px`) over the bar's border to merge into the terminal. Same-height is the point: earlier attempts made the active tab taller/shifted (float-chip, then `::after` bridge), which pushed its text off the shared baseline ‚Äî rejected both. The `+` is borderless and parked far right via `margin-left: auto`, and moved in the JSX to render **after** the dormant/ghost tabs so it's never stranded mid-bar.
- 2026-07-18 ‚Äî `SessionMeta.messageCount` (the sidebar emptiness gate in `scan.ts`) counts real messages across the **whole main timeline**, not the linearized active branch. Why: a Claude session mid-write can transiently linearize to 0 countable messages (a compaction/summary boundary breaks the `parentUuid` walk), which dropped a live session out of the sidebar for the length of its write burst (~20s, stretched by the watcher's 1s trailing debounce being continually reset). Whole-timeline count is monotonic and never dips to 0. Codex already counted whole-file, so unaffected. Left the watcher debounce alone (a maxWait would re-scan all sessions repeatedly during heavy writes).
- 2026-07-17 ‚Äî Dogfooding runs the **packaged app** (`npm run dist:install` ‚Üí /Applications/Chewo.app via electron-builder), not `electron-vite preview` ‚Äî preview shares `out/` with dev and gets clobbered by rebuilds. Local builds skip keychain signing (`identity: null`, duplicate Apple Development certs make it ambiguous); ad-hoc is fine locally. Dev instance uses `userData` suffix `-dev` so dev and installed app never fight over projects.json; the installed app inherits the original `chewo` data dir.
- 2026-07-18 ‚Äî Transcript‚Üínote structuring pins **`--model sonnet`** in `claude -p` (src/main/structure.ts). Fast/cheap enough per lesson, strong enough for faithful sectioning; revisit toward Haiku (speed) or Opus (depth) after real lectures.
- 2026-07-18 ‚Äî Notes hierarchy vocabulary: **Subject ‚Üí Topic ‚Üí Lesson** (UI copy only; code keeps Note* identifiers, mirroring the "session" vocab decision). A lesson is the cohesion unit: typed notes and dictations live in one file.
- 2026-07-18 ‚Äî Dictation **appends to the selected lesson** (gated on subject+topic+lesson), never creates a note per recording. Structuring prompt gets the lesson's current body and must extend without repeating, no per-chunk summaries; transcript goes to the lesson's append-only `.raw.md` twin before Claude runs; a failed pass appends the raw transcript instead. Appends flow through the open editor (pendingAppend) so typing during recording is never clobbered.
- 2026-07-18 ‚Äî Live transcript paragraphs come from **segment silence gaps** (>1.75 s) in the sidecar ‚Äî no LLM in the live path. While recording, the lesson pane is Note | Live transcript tabs; type and read simultaneously.
