# Decisions

- 2026-07-17 — Worktree isolation is **opt-in per terminal**, not the default. Martin deliberately runs multiple agents in one checkout; the main checkout keeps the dev servers/ports. Rejected: default-isolation, concurrency badge, per-worktree dev servers / port offsetting.
- 2026-07-17 — Worktrees are headless scratch buffers: merge back into the main checkout to see anything running. Setup command (per-project, user-authored) instead of auto-copying gitignored files — secret-leak risk.
- 2026-07-17 — Permission mode is a **per-section Chewo setting** (`--permission-mode` / `--ask-for-approval` per spawn), not `permissions.defaultMode` in `~/.claude/settings.json`. Why: the global route widens every repo on the machine including fresh untrusted clones, and can't cover Codex without hand-writing `config.toml` (forbidden — live-rewritten). Diagnosis: neither CLI remembers the mode you flip to, so Chewo's many fresh sessions always started at the default.
- 2026-07-17 — Home is a first-class section, not a fallback. Sessions map to their owning section only; nothing inherits the currently-selected project.
- 2026-07-17 — Notes workflow (SPEC-NOTES.md): STT runs in a **local sidecar process** with a pluggable JSON-stdio protocol (WhisperKit CLI first, parakeet-mlx later). Rejected: whisper.cpp in-process (rewrites proven confirm-and-seek logic, blocks Parakeet A/B), cloud STT (cost, audio leaves machine).
- 2026-07-17 — Notes are **markdown files + folders** (Subject/Topic/note.md) under a configurable root. Rejected: projects.json blob (transcripts bloat it), SQLite (breaks the no-DB/no-cache design).
- 2026-07-17 — Notes intelligence (structuring + Q&A) runs via **headless `claude -p`**, Q&A scoped by setting cwd to the subject/topic folder with Read/Grep/Glob only. Rejected: direct Anthropic API (second billing/key path), Ollama (weak long-transcript structuring).
- 2026-07-17 — Structuring runs **once on stop** over the full transcript; taxonomy is **fully manual** (user names subject + topic before capture); raw **transcript is kept, audio discarded** after transcription.
- 2026-07-18 — Umbrella term for one item in the tab bar is **"session"** (vocabulary only, no code rename yet): what you create/focus/close. Its layers keep their code-level names — **tab** (UI chip), **terminal** (`termId`, pty runtime), **conversation** (`sessionId`, the CLI transcript a session may be bound to; shell sessions have none). Rejected "tab" (pure chrome, says nothing about the work) and "terminal" (undersells the agent/dormant layers).
- 2026-07-18 — Closing the focused session focuses its **left neighbour** in the same section (fall back to the right, then empty). Previously dropped straight to the empty state even with other sessions open.
- 2026-07-18 — Tab bar is a **faithful Chrome replica** (`min-height: 38px`): borderless tabs, packed with 1px `::before` **dividers** between them (hidden either side of the active/hovered tab), not gaps. All tabs are the **same height** — the focused tab (`terminal-tab-active`, exists only when `view.kind === 'terminal'`) merely fills with the terminal surface (`--bg`), keeps rounded-top corners, and drops 1px (`margin-bottom: -1px`) over the bar's border to merge into the terminal. Same-height is the point: earlier attempts made the active tab taller/shifted (float-chip, then `::after` bridge), which pushed its text off the shared baseline — rejected both. The `+` is borderless and parked far right via `margin-left: auto`, and moved in the JSX to render **after** the dormant/ghost tabs so it's never stranded mid-bar.
- 2026-07-18 — `SessionMeta.messageCount` (the sidebar emptiness gate in `scan.ts`) counts real messages across the **whole main timeline**, not the linearized active branch. Why: a Claude session mid-write can transiently linearize to 0 countable messages (a compaction/summary boundary breaks the `parentUuid` walk), which dropped a live session out of the sidebar for the length of its write burst (~20s, stretched by the watcher's 1s trailing debounce being continually reset). Whole-timeline count is monotonic and never dips to 0. Codex already counted whole-file, so unaffected. Left the watcher debounce alone (a maxWait would re-scan all sessions repeatedly during heavy writes).
- 2026-07-17 — Dogfooding runs the **packaged app** (`npm run dist:install` → /Applications/Chewo.app via electron-builder), not `electron-vite preview` — preview shares `out/` with dev and gets clobbered by rebuilds. Local builds skip keychain signing (`identity: null`, duplicate Apple Development certs make it ambiguous); ad-hoc is fine locally. Dev instance uses `userData` suffix `-dev` so dev and installed app never fight over projects.json; the installed app inherits the original `chewo` data dir.
- 2026-07-18 — Transcript→note structuring pins **`--model sonnet`** in `claude -p` (src/main/structure.ts). Fast/cheap enough per lesson, strong enough for faithful sectioning; revisit toward Haiku (speed) or Opus (depth) after real lectures.
- 2026-07-18 — Notes hierarchy vocabulary: **Subject → Topic → Lesson** (UI copy only; code keeps Note* identifiers, mirroring the "session" vocab decision). A lesson is the cohesion unit: typed notes and dictations live in one file.
- 2026-07-18 — Dictation **appends to the selected lesson** (gated on subject+topic+lesson), never creates a note per recording. Structuring prompt gets the lesson's current body and must extend without repeating, no per-chunk summaries; transcript goes to the lesson's append-only `.raw.md` twin before Claude runs; a failed pass appends the raw transcript instead. Appends flow through the open editor (pendingAppend) so typing during recording is never clobbered.
- 2026-07-18 — Live transcript paragraphs come from **segment silence gaps** (>1.75 s) in the sidecar — no LLM in the live path. While recording, the lesson pane is Note | Live transcript tabs; type and read simultaneously.
- 2026-07-19 — STT sidecar is built by an **incremental guard** (`scripts/ensure-sidecar.mjs`, mtime of the binary vs `Sources/`+manifests) wired into `predev` (`--soft`: warns, dev still runs) and `predist` (strict: fails the release), and **packaged via electron-builder `extraResources`** → `Resources/bin/chewo-stt-whisper` (the path `sidecarPath()` checks when `app.isPackaged`). Why: the packaged app silently shipped *without* the sidecar (no extraResources rule existed) and dev relied on remembering `npm run build:stt`. The guard skips instantly when the binary is current, so `predev`/`predist` are cheap to always run. First build is slow (downloads WhisperKit) — the script prints a heads-up. Sidecar is self-contained (system libs + OS Swift runtime only), so copying just the executable suffices.
- 2026-07-18 — **UI overhaul direction (design/):** planned via 5 persona passes (Art Director→Visual→Design Systems→Motion; "twice-as-premium" cut — personal daily driver). Original "Kiln" warm-terracotta base was **rejected** ("mustard brown/ugly"); pivoted to **Graphite base (neutral gray) + Emerald accent** `hsl(158 52% 48%)`. "Green = accent only": running/live moved to cyan `#34C9D6`, recording to red `#E2574B`. Source badges keep brand colors (CC terracotta, CX vivid blue). Variants: serif ON, Codex vivid, grain ON, terminal full-bleed. Sidebar top = switcher-on-own-row + subtle (non-filled) "+ New session ▾" CTA. Token architecture from `design/04` stands; only hex values change. Canonical spec: `design/06-chosen-direction.md`. Planned: in-app Appearance settings page (curated base+accent pickers) — needs runtime re-theming of xterm/CodeMirror/window (they read JS, not CSS vars).
- 2026-07-19 — **Appearance settings** (in-app, ⌘, or gear by the workflow switcher) delivered: user-editable Base, Accent (+ secondary/tertiary), 16 terminal ANSI colors, 13 editor syntax colors. Persisted at `userData/settings.json` (`settings:load/save`, same shape as projects.json). Applies **live** — CSS `:root` primitive overrides re-theme the whole UI; xterm mutates `options.theme`; CodeMirror gets a rebuilt `theme` prop; the native window follows via `win.setBackgroundColor`. Single source of truth for the color math is `src/shared/appearance.ts` (dependency-free, imported by both main and renderer); `src/shared/colors.ts` deleted. **Whole neutral surface ramp + text ramp + rim are derived from Base** (`deriveSurfaces`/`deriveTextRamp`/`deriveRim`): a neutral base reproduces the stock graphite+cream *exactly*, a saturated base pulls surfaces/text/rim toward its hue (full by ~40% sat). **Status (danger/warning) and source badges (Claude/Codex) stay fixed** — honors "green = accent only" + badges-are-brand-identity from the design canon; deliberately not exposed. Codebase was already ~99% tokenized (only 2 stray black scrim/shadow values found, now `--scrim`/`--shadow-toast`); components carry zero hardcoded color.
- 2026-07-19 — **Appearance settings UX** reworked from a stacked modal into a **full-screen view** (overlays `.main-panel`, sidebar stays; `position:absolute inset:0` — `.main-panel` made `position:relative`) with **4 tabs**: Presets (landing) · App · Terminal · Editor. Each category tab is **two-column: controls left, a live preview mockup right** that updates as you edit. Previews are **real live components**, not images — App preview reads CSS `:root` tokens; Terminal/Editor previews read the in-flight `appearance` prop (their colors are JS, not CSS vars) and derive canvas/text via `deriveSurfaces`/`deriveTextRamp`, matching the real xterm/CodeMirror themes. **Presets** (`src/shared/presets.ts`, ~6 dark: Graphite Emerald=default, Midnight, Nocturne, Ember, Nightshade, Rosé) apply a whole `AppearanceSettings`; `matchPreset()` shows the active card or "Custom". High-impact fields (Base + 3 accents) get a **curated quick-pick swatch row** (`CURATED_BASES`/`CURATED_ACCENTS`) on top of picker+hex. Light presets deferred (derive math assumes a dark base). Components in `src/renderer/src/components/settings/`.
- 2026-07-19 — **Todo workflow spec drafted** (`SPEC-TODOS.md`): Kanban (Blocked|Todo|In Progress|Done) as the **third workflow segment**; boards per project + a General board; voice control via **system-wide toggle hotkey** (globalShortcut has no key-up → no push-to-talk) with an always-on-top HUD, existing STT sidecar, and a **headless Sonnet interpreter** (`claude -p --model sonnet --bare --output-format json --json-schema`) that infers the target board from the utterance (fallback General) and has **full command breadth** (add/move/edit/delete) executed immediately with an undo toast. Storage: **global `~/.chewo/todos/<scope>/board.json`** (greppable, survives app-data resets, agent-readable) — not userData, not in-repo. STT cold start solved by **overlap-capture-with-load + 15-min idle unload** (revised same day from always-resident launch prewarm — 1–1.5 GB held for once-every-2h usage was the wrong trade): hotkey opens the mic immediately, sidecar buffers audio while the model loads in parallel (~0 perceived latency), new `prewarm`/`unload` protocol commands, notes recording view prewarms on open. Single shared model (`large-v3-turbo`); a smaller command-only model rejected (`base.en` measured ~90% on Martin's accented speech; misses land in card titles — `small.en` is the floor if ever needed). Verified 2026-07-19: `claude -p` has **no resident/multi-turn stream-json mode** — per-call cold start is unavoidable; `--json-schema` (enforced `structured_output`) works, but **`--bare` breaks keychain auth on CLI 2.1.215** ("Not logged in") and was dropped — retest on CLI updates; direct Anthropic API stays the fallback. Measured interpreter round trip ~4–5 s. **Haiku benchmarked 2026-07-19 as a speed candidate and rejected**: 3 runs each via `claude -p --json-schema`, haiku ~8.5–8.8 s API vs sonnet ~4.5–5.2 s (same correct output) — slower despite being the smaller model, so sonnet stays. MCP tools via context-bridge deferred to T3.
- 2026-07-19 — **Notes tab + terminal bright-colors** in Appearance. Added a 5th settings category **Notes** (`NotesColors {heading, link, code, quote}` in appearance.ts, in every preset, deep-merged in settings.ts) themable independently of the app accent. Applied as `--notes-*` CSS vars (set by applyAppearance) consumed **only by `.notes-md-preview`** so the shared `.message-markdown` (transcript/chat/capabilities) is untouched; the notes overrides fall back to app tokens (`var(--notes-link, var(--accent))`) and win over `.message-markdown` by source order (equal specificity). Live `NotesPreview` mock (mini lesson) reads `appearance.notes` inline. Note: the notes **code editor** (edit mode) is already themed by the Editor/syntax tab; the Notes tab themes the rendered **preview** — hint copy updated to say so. Terminal preview now splits the ANSI strip into labeled **Normal/Bright** rows and demonstrates bright colors in sample output (brightBlack = metadata grey, bright green/white = bold) so editing them is visible — no separate mock needed.
- 2026-07-19 — **Local data placement rule (three tiers).** Litmus test: *would the user open this file outside Chewo?* (1) **User documents → visible folder**: notes stay in a visible, configurable root; default flipped to `~/Documents/Chewo Notes` (rides iCloud Documents sync; installs with an existing `~/ChewoNotes` keep it — `DEFAULT_NOTES_ROOT` in `src/main/notes.ts`). (2) **Machine-managed, agent-reachable data → `~/.chewo/`**: todos (`~/.chewo/todos/`) and future caches — greppable, survives app-data resets, not hand-edited. (3) **Disposable app machinery → Electron userData**: settings.json, projects.json. **In-repo storage vetoed**: worktree agents would fork the data, dirty `git status`, risk commits; General board / cross-project subjects have no repo. Folders are created lazily on first use, never at install.
- 2026-07-20 — **PDF viewing uses Chromium's built-in viewer, not a bundled pdf.js.** `.pdf` became a third `readFile` kind alongside text/image (`src/main/file-explorer.ts`), returning a base64 `data:application/pdf` URL capped at 25 MB; `FileEditor` converts it to a **blob: URL** and renders an `<iframe>`. Chose this over `pdfjs-dist` — zero dependency, and the free viewer chrome (zoom, page thumbnails, find, print, download) beats anything worth hand-building. Requires `webPreferences.plugins: true` (defaults false; without it the iframe is blank). **Gotcha: the viewer refuses a `data:` PDF URL, and a blob minted from an opaque origin renders as an empty viewer** — toolbar paints, page area stays blank. Verified 2026-07-20 against Electron 43 with a throwaway harness: `data:text/html` host → `blob:null/…` → blank; `http://127.0.0.1` and `file://` hosts → renders. Both of the app's real origins (dev server, packaged `loadFile`) are fine, so this only bites if the renderer is ever moved to an opaque origin. Side effect: the viewer's title bar shows the blob UUID, not the filename — the file chip strip above it already names the file.
- 2026-07-21 — **System-audio capture via Core Audio process tap, not ScreenCaptureKit or BlackHole.** Notes gained a second record mode: **device + mic** (`source:"mix"` in the sidecar protocol) for online lessons and meetings. Chose `AudioHardwareCreateProcessTap` (macOS 14.2+) + a private **aggregate device** holding the default mic subdevice and the tap with **drift compensation on** — the HAL reconciles the two clocks, so one IOProc receives both in sync (no hand-rolled ring-buffer sync); channels are summed to mono and resampled 48k→16k through one stateful `AVAudioConverter`. ScreenCaptureKit rejected (screen-recording TCC prompt for an audio-only need); BlackHole rejected (user-installed driver, hijacks output routing). Permission: `NSAudioCaptureUsageDescription` is its own TCC category — embedded in the bare CLI via `__TEXT,__info_plist` linker section (AudioTee pattern) + `mac.extendInfo`; ad-hoc signing means the grant can re-prompt when the sidecar binary changes (accepted for dogfooding). Capture **source is orthogonal to note style**: the session recorder picks both independently (mic for in-person, device+mic for online × lecture/meeting) — style only switches the structuring prompt (meeting prompt adds Decisions / Action items); the Mic toolbar button stays as one-click dictation (mic+lecture). Same session, the engine got a **rolling buffer**: confirmed audio is dropped from a lock-guarded `SampleBuffer` and confirmed text is built incrementally, so hour-long recordings stay flat on memory/CPU (previously the full sample array was copied every 750 ms — O(n²)).
- 2026-07-23 — **Git panel v1 is read-only** (status / diff / history — the "GitHub Desktop glance" without GitHub): no stage/commit/push until the read layer proves itself, so the app can never corrupt a repo mid-agent-operation. `src/main/git.ts` shares `runGit` with worktrees.ts, parses only plumbing formats (`--porcelain=v2 -z`, `--numstat -z`, `%x1f` logs — locale/quote-proof, space-safe paths), and never caches: agents mutate repos from outside, every call re-reads. Live updates via one recursive chokidar watcher per subscribed root with `node_modules` ignored and `.git` internals filtered to HEAD/index/refs/packed-refs (objects churn never wakes the renderer); watcher refuses `$HOME` itself, so git features are project/worktree-scoped — Home shows a disabled toggle. Worktree session tabs get a dirty-count pill from **15 s polling** instead (a recursive watcher per worktree tab wasn't worth it). UI: toggle (⌘⇧G) sits beside the file-tree toggle with an always-on dirty-count badge, panel is mutually exclusive with the file tree, diff renders as a layer over the terminal like FileEditor (unified only; `@codemirror/merge` split view deferred). Diff add-green is hue 135, deliberately not the emerald accent, per design/06 "green = accent only".
