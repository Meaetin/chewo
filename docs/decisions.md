# Decisions

- 2026-07-17 — Worktree isolation is **opt-in per terminal**, not the default. Martin deliberately runs multiple agents in one checkout; the main checkout keeps the dev servers/ports. Rejected: default-isolation, concurrency badge, per-worktree dev servers / port offsetting.
- 2026-07-17 — Worktrees are headless scratch buffers: merge back into the main checkout to see anything running. Setup command (per-project, user-authored) instead of auto-copying gitignored files — secret-leak risk.
- 2026-07-17 — Permission mode is a **per-section Chewo setting** (`--permission-mode` / `--ask-for-approval` per spawn), not `permissions.defaultMode` in `~/.claude/settings.json`. Why: the global route widens every repo on the machine including fresh untrusted clones, and can't cover Codex without hand-writing `config.toml` (forbidden — live-rewritten). Diagnosis: neither CLI remembers the mode you flip to, so Chewo's many fresh sessions always started at the default.
- 2026-07-17 — Home is a first-class section, not a fallback. Sessions map to their owning section only; nothing inherits the currently-selected project.
- 2026-07-17 — Notes workflow (SPEC-NOTES.md): STT runs in a **local sidecar process** with a pluggable JSON-stdio protocol (WhisperKit CLI first, parakeet-mlx later). Rejected: whisper.cpp in-process (rewrites proven confirm-and-seek logic, blocks Parakeet A/B), cloud STT (cost, audio leaves machine).
- 2026-07-17 — Notes are **markdown files + folders** (Subject/Topic/note.md) under a configurable root. Rejected: projects.json blob (transcripts bloat it), SQLite (breaks the no-DB/no-cache design).
- 2026-07-17 — Notes intelligence (structuring + Q&A) runs via **headless `claude -p`**, Q&A scoped by setting cwd to the subject/topic folder with Read/Grep/Glob only. Rejected: direct Anthropic API (second billing/key path), Ollama (weak long-transcript structuring).
- 2026-07-17 — Structuring runs **once on stop** over the full transcript; taxonomy is **fully manual** (user names subject + topic before capture); raw **transcript is kept, audio discarded** after transcription.
- 2026-07-18 — Umbrella term for one item in the tab bar is **"session"** (vocabulary only, no code rename yet): what you create/focus/close. Its layers keep their code-level names — **tab** (UI chip), **terminal** (`termId`, pty runtime), **conversation** (`sessionId`, the CLI transcript a session may be bound to; shell sessions have none). Rejected "tab" (pure chrome, says nothing about the work) and "terminal" (undersells the agent/dormant layers).
- 2026-07-18 — Closing the focused session focuses its **left neighbour** in the same section (fall back to the right, then empty). Previously dropped straight to the empty state even with other sessions open.
- 2026-07-18 — Tab bar is **focus-driven, Chrome-style**: at rest every tab + the `+` are small centred floating chips (`align-items: center`, fully rounded, `--bg-hover`) connected to nothing. Only the focused session's tab (the `terminal-tab-active` class, which exists solely when `view.kind === 'terminal'`) recolours to the terminal surface (`--bg`), `align-self: flex-end` + taller padding to grow down to the rail, rounded top only, and `margin-bottom: -1px` over the bar's `border-bottom` so tab + terminal merge into one piece. On a transcript/empty/capabilities view nothing is connected. Inverted from the original (active was the lightest, floating chip).
- 2026-07-18 — `SessionMeta.messageCount` (the sidebar emptiness gate in `scan.ts`) counts real messages across the **whole main timeline**, not the linearized active branch. Why: a Claude session mid-write can transiently linearize to 0 countable messages (a compaction/summary boundary breaks the `parentUuid` walk), which dropped a live session out of the sidebar for the length of its write burst (~20s, stretched by the watcher's 1s trailing debounce being continually reset). Whole-timeline count is monotonic and never dips to 0. Codex already counted whole-file, so unaffected. Left the watcher debounce alone (a maxWait would re-scan all sessions repeatedly during heavy writes).
- 2026-07-17 — Dogfooding runs the **packaged app** (`npm run dist:install` → /Applications/Chewo.app via electron-builder), not `electron-vite preview` — preview shares `out/` with dev and gets clobbered by rebuilds. Local builds skip keychain signing (`identity: null`, duplicate Apple Development certs make it ambiguous); ad-hoc is fine locally. Dev instance uses `userData` suffix `-dev` so dev and installed app never fight over projects.json; the installed app inherits the original `chewo` data dir.
